PROJECT: Shiksha Vatika Public School - Official Website
TECH STACK: HTML5, CSS3 (BEM-like structure), Vanilla JavaScript (ES6+)
CDNS: Google Fonts, Font Awesome (or Material Icons), Google Maps embed, Crisp/WhatsApp widget CDN
LANGUAGES: English + Hindi (language switcher)
RESPONSIVE: Mobile-first, breakpoints: 360px, 768px, 1024px

GOALS:
- Premium, trust-inspiring one-page site + “Admissions” form page
- Fast (<= 1s TTFB not required but target under 2s load on 3G)
- Accessible (WCAG AA basics: alt tags, semantic tags, keyboard nav, color contrast)
- SEO basics: meta tags, schema for Organization, sitemap.xml, robots.txt
- Easy to update (single JS config file for content copy + translations)
- Contact actions: direct phone links, WhatsApp button, email contact form (POST to endpoint or integrate with Google Forms/Sheets)
- Deployable to static hosts: Netlify, Vercel, GitHub Pages, or any shared hosting.

DESIGN SPECS
- Fonts: Poppins or Montserrat for headings, Roboto for body (via Google Fonts).
- Colour palette (derived from images):
  - Primary deep blue: #083a8b (hero background)
  - Accent yellow/gold: #f5b400
  - Green highlight: #1e8f4f
  - Dark navy for text: #0b2136
  - Soft white background: #f7f9fb
- Buttons: rounded 8px, subtle shadow, hover transform -2px
- Card radius: 12px, 2px soft shadow on desktop
- Grid: CSS Grid for layouts, Flexbox for lists
- Images: use srcset for responsive images and lazy load

LANGUAGE / I18N
- Use a single JS translation file per language (`content-copy-en.json`, `content-copy-hi.json`)
- Language selection updates visible text and `lang` attribute on `<html>`
- Persist language in `localStorage`
- Translate all buttons, headings, form labels, success messages and footer.

FORMS / BACKEND (no server required; static-friendly options)
1) Preferred (no-code): Submit form to Google Forms/Sheets endpoint (AJAX) — provide docs on how to map fields.
2) Alternative: Send POST via free form service (Formspree, Getform) — configure single endpoint in `config.js`.
3) Option: Create a simple PHP form handler (if hosting supports PHP) sending email, saving attachments.
